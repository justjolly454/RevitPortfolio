01
Dimensioning Tool
Revit Macros ✅ Active
This tool automates the placement of end to end dimensions on valves placed in the Revit project, thus reducing repetitive manual effort and improving the QC process.
▼
Process Before This Tool

Following are the steps involved in the manual method:
1. Activate the Aligned Dimension Tool
On the Annotate tab, select Aligned Dimension to begin placing dimensions.
2. Select the First Reference Point
Click on the left endpoint of the valve to establish the first dimension reference.
3. Select the Second Reference Point
Click on the right endpoint of the valve to define the second reference.
4. Place the Dimension
Move your cursor to the desired location and click to place the aligned dimension.
5. Repeat for Additional Valve Sizes
Repeat Steps 2–4 for each remaining valve size that requires dimensioning.

Before & After
⏱ Manual Method
⚡ Automated Method
Implications & Impact
⏱ For ~350 instances, 25 minutes → 10 seconds    ~99% time reduction per project
📈 Efficiency Gain Since the tool is selection based, it works on any number of instances in seconds - be it 7 or 700 instances.
🎯 Error Reduction Reduces the possibility of manual error due to its repetitive nature
💼 Business Value The more instances you run, the more time you save.
⚙️ Tech Used VB.NET, Revit Macros, Copilot
02
SaveAs Families
Revit Macros ✅ Active
This tool saves all selected families placed in the project into the desired location all at once.
▼
Process Before This Tool

Following are the steps involved in the manual method:
1. Open the Family for Editing
Select the desired family instance within the Revit project (.rvt) and click Edit Family from the ribbon to enter the Family Editor environment.
2. Access the Save As Dialog
In the Family Editor, navigate to File and select Save As > Family to open the Save As dialog box.
3. Specify the Save Location
Browse to the desired destination folder where the family file (.rfa) is to be saved.
4. Retain the Family Name and Save
Ensure the file name remains unchanged, then click Save to export the family as a standalone .rfa file.
5. Repeat for Additional Families
Return to the Revit project environment and repeat Steps 1 through 4 for each remaining family that needs to be saved individually.

Before & After
⏱ Manual Method
⚡ Automated Method
Implications & Impact
⏱ For ~50 families, 16 minutes → 10 seconds    ~97% time reduction
📈 Efficiency Gain Since the tool is selection based, it works on any number of instances in seconds - be it 7 or 700 instances.
🎯 Error Reduction Reduces fatigue and errors in filename while browsing
💼 Business Value The more instances you run, the more time you save.
⚙️ Tech Used VB.NET, Revit Macros, Copilot
03
Family Configurator
Revit Plugins 🔄 Ongoing
This tool exposes the parameters of parametric families directly within the project environment — allowing users to configure and place custom family instances without ever leaving Revit.
▼
Process Before This Tool

Following are the steps involved in the traditional method:
1. Locate the Parametric Family
Navigate to the shared library location and find the required parametric family file (.rfa).
2. Create a Local Copy
Duplicate the file into a local folder and rename it to reflect the desired configuration — to avoid overwriting the original.
3. Load and Open Edit Type
Load the renamed family into the Revit project, select a placed instance, and open the Edit Type dialog from the Properties panel.
4. Modify Parameters Manually
Change the required parameter values within the Edit Type dialog to achieve the desired family configuration.
5. Cross-Reference the Drawing
Keep the reference drawing open separately to identify which parameter corresponds to which dimension — switching back and forth throughout the process.

Tool Demo
⚡ Automated Method
Implications & Impact
⏱ Multi-step external process → Single in-project workflow    Significant reduction in configuration time and reference errors
📈 Efficiency Gain Family selection, parameter editing, and instance placement all happen inside one interface — no file navigation, no local copies, no switching between windows.
🎯 Error Reduction The built-in reference drawing image eliminates the need to cross-reference external documents. Parameters are labelled in context — reducing the chance of editing the wrong value.
💼 Business Value Faster family configuration directly accelerates content creation and modelling workflows.
⚙️ Tech Used Revit API, VB.NET, Copilot
04
Lookup Generator
Excel Macros 🔄 Ongoing
This tool extracts submittal data from a template Excel file and generates a Revit-ready lookup table (.CSV) automatically — eliminating manual data entry and transcription errors.
▼
Process Before This Tool

Following are the steps involved in the manual method:
1. Locate an Existing Lookup Table
Open an existing .CSV lookup table that closely matches the required format, to use as a reference for the correct header structure.
2. Match Headers Manually
Compare the submittal data against the lookup table headers and manually rearrange or rename columns to align with Revit's expected format.
3. Enter Sizes and Corresponding Values
Key in each size and its corresponding parameter values row by row, referencing the submittal document alongside the lookup table.
4. Save and Load into Revit
Save the file as a .CSV and load it into Revit to verify the data is read correctly.

Process After This Tool

Following are the steps involved in the automated method:
1. Get the data
Open excel and extract the submittal data using Data >> Get Data >> From Other Sources >> From Web
2. Format the extracted data
Copy the extracted data to the macro enabled file Lookup Table Generator to format the unwanted data.
3. Match the headers
Formatted data need to have pre-requisite headers.
4. Run the macro - Lookup file ready!
Run the Lookup Generator macro, give the file a path and a name. Your .csv file is ready with required headers and metric values.

Implications & Impact
📝Manual entry errors → Near zero
📈 Efficiency Gain Extraction and formatting are handled by macros — the user only verifies, not re-types.
🎯 Error Reduction Eliminates transcription mistakes that occur when manually matching headers and entering values row by row.
💼 Business Value Fewer errors at the content creation stage means fewer rework cycles downstream — saving time at every point the data is used.
⚙️ Tech Used Excel Macros, Copilot
05
Weekly Report Generator
Excel Macro ✅ Active
This tool automatically compiles equipment creation requests across all projects into a formatted Word report — ready for the weekly manager meeting with just one macro run.
▼
Process Before This Tool

Following are the steps involved in the manual method:
1. Pull Images from Project Files
Open each project file individually and extract the images of completed equipment creation requests one by one.
2. Build the Word Document
Create a new Word file and insert each image alongside its corresponding equipment name and project name manually.
3. Format for Readability
Adjust layout, spacing, image sizes, and text formatting repeatedly to make the report compact and presentable.
4. Name the File
Rename the final document to reflect the meeting date before sharing or presenting.
5. Repeat Every Week
Rebuild the entire document from scratch the following week — same process, same effort.

Tool Demo
⚡ Automated Method
Implications & Impact
⏱ ~1 hour → Under 5 minutes    ~92% time reduction · Every single week
📈 Efficiency Gain Images are organised into dated folders progressively throughout the week. On meeting day, one macro run assembles everything — formatting, naming, and all.
🎯 Error Reduction Eliminates the risk of missing an equipment entry, inconsistent formatting, or an incorrectly named file — errors that are easy to make when rebuilding a document manually under time pressure before a meeting.
